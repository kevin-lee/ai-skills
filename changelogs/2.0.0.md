## [2.0.0](https://github.com/kevin-lee/ai-skills/issues?q=is%3Aissue%20is%3Aclosed%20milestone%3Am4) - 2026-04-01 🤥

## Breaking Changes

### Consolidate `manage` and `remove` into a unified `remove` command (#41)

1. The current `manage` command is just to remove skills in interactive mode.
2. The current `remove` lacks agent and location-specific removal options. It only removes whatever is found first.

So this release consolidates `manage` and `remove`, and removes `manage`.
So `remove` has interactive mode as well.
In addition, it adds options to specify agent and location in non-interactive mode.


* `manage`'s interactive mode is moved to `remove`, and the `manage` command is removed.
* `remove` has parameters to specify `agent` and location (`project` and/or `global`).

```
aiskills remove SKILL_NAME --agent agents,in,csv (--project|--global|--project --global)
```

#### Examples

```
aiskills remove                                           # Interactive multi-select
aiskills remove commit --agent claude --project           # Project, Claude
aiskills remove commit --agent claude --global            # Global, Claude
aiskills remove commit --agent claude --project --global  # Both scopes, Claude
aiskills remove commit --agent all --project              # Project, all agents
aiskills remove commit --agent all --project --global     # Remove everywhere
aiskills remove commit                                    # ERROR: must specify --project/--global
aiskills remove commit --project                          # ERROR: must specify --agent
```
***

### Replace `--all-agents` parameter with `--agent all` for the commands supporting `--all-agents` (#43)

Just like the `remove` command, `--agent all` is much simpler and fewer parameters to remember, so `--all-agents` has been replaced with `--agent all`.
***

### Require explicit `--project`/`--global` flags instead of implicit both-scopes behavior in non-interactive mode (#47)

For non-interactive mode
```
aiskills list --project --global
```
now shows both project and global skills

```
aiskills install SKILL --agent AGENT --project --global
```
now installs SKILL for both project and global.

***

## Improvements

### Add Location Selection to Sync Interactive Mode (#44)

Previously, sync interactive mode did not prompt for location selection. Now it follows this flow:

1. **Select source location**: `project` | `global` (singleChoice)
2. **Select source agent**: filtered to agents with skills in the selected source location, with accurate counts
3. **Select target location**: `project` | `global` | `both` (singleChoice)
4. **Select target agent(s)**: all agents except source agent (multi-select)

> **NOTE:**
> The actual interactive flow was replaced by ['Skill selection should be added to sync's interactive mode (#52)'](#user-content-skill-selection-should-be-added-to-syncs-interactive-mode-52).

***

### Improve `read` command with interactive mode and more non-interactive mode parameters (#49)

#### Interactive Mode

1. Single selection of location: `project` or `global` or both
2. Multiple selection of all agents available in the location selected in 1.
3. List of all available skills from the previous selections. The skill name should also include the typical `(location, Agent): path` information.

#### Non-Interactive Mode

```
aiskills read SKILL_NAME1 SKILL_NAME2 --agent claude,cursor --project --global
aiskills read SKILL_NAME1 --agent claude --project
aiskills read SKILL_NAME1 --agent claude --global
```
> **NOTE:**
> `--prefer` has been removed. The explicit `--project`/`--global` flags now replace the previous location preference behavior.

***

### Add location support for `sync`'s non-interactive mode (#51)

```
aiskills sync                                                              # Interactive wizard
aiskills sync --from project:claude --to cursor --project                  # All skills, project -> project
aiskills sync --from global:claude --to cursor,windsurf --global           # All skills, global -> global
aiskills sync --from project:claude --to all --project --global            # All skills, project -> both
aiskills sync commit --from global:claude --to cursor --project --global   # One skill, global -> both
aiskills sync --from project:universal --to copilot --global -y            # Skip confirmation prompts
```
***

### Skill selection should be added to `sync`'s interactive mode (#52)

The interactive flow is now:
1. Select source location (`project`/`global`)
2. Select source agent
3. **Select skills to sync (new - multi-select, all pre-selected) <== 💥**
4. Select target location (`project`/`global`/both)
5. Select target agent(s)
***

### `sync`'s non-interactive mode should support syncing multiple selected skills (#54)

You can now specify multiple skill names as positional arguments to sync a subset of skills instead of syncing all:

```
aiskills sync skill-1 skill-2 skill-3 --from project:claude --to codex,gemini --project
```
