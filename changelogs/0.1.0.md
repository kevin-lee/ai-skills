## [0.1.0](https://github.com/kevin-lee/ai-skills/issues?q=is%3Aclosed%20milestone%3Am1) - 2026-03-25

## Done
* Initial release of ai-skills

### 🚀 Features
- **Native CLI Tool**: Built with Scala 3 and Scala Native for standalone binary execution without JVM or Node.js runtime
- **Multi-Agent Support**: Support for Universal, Claude, Cursor, Codex, Gemini, and Copilot agents
- **Skill Management Commands**:
  - `list` - List all installed skills across agents
  - `install` - Install skills from Git repos, GitHub shorthand, or local paths
  - `read` - Read skill content for AI agent consumption
  - `update` - Update installed skills from original sources
  - `sync` - Sync skills between different agent directories
  - `manage` - Interactive skill management interface
  - `remove` - Remove individual skills
- **Skill Format**: YAML frontmatter-based SKILL.md format with name and description metadata
- **Directory Structure**: Support for both project-local and global skill directories with priority-based resolution
- **Interactive Selection**: Multi-select prompts for skill installation and management using cue4s

### 🏗️ Architecture
- **Modular Design**: Split into `ai-skills-core` (domain logic) and `ai-skills-cli` (CLI interface)
- **Type Safety**: Comprehensive ADTs for error handling, skill sources, and agent types
- **Functional Programming**: Immutable data structures and pure functions throughout
- **Testing**: Extensive test coverage using Hedgehog property-based testing and MUnit

### 🔧 Technical Implementation
- **Build System**: SBT with Scala Native compilation to standalone binaries
- **Dependencies**: Minimal external dependencies using os-lib for file operations and decline/cue4s for CLI
- **Code Quality**: Scalafmt formatting, Scalafix linting, and comprehensive error handling
- **Security**: Safe YAML parsing with ReDoS protection and input validation

### 📦 Project Structure
```
modules/
├── ai-skills-core/  # Domain types, skill discovery, YAML parsing, metadata I/O
└── ai-skills-cli/   # CLI entry point and commands
```

### 🔄 Development History
- **Project Foundation**: Initial setup with build configuration, licensing, and code formatting standards
- **Core Implementation**: Complete skill management system with CLI commands and utilities
- **Code Refactoring**: Applied Scala 3 brace syntax, removed mutable state, improved functional patterns
- **UI Polish**: Cleaned up interactive prompts and refined CLI argument handling
- **CI/CD**: GitHub Actions workflows for builds, checks, and automated releases
- **Quality Assurance**: Comprehensive test suite with property-based testing and edge case coverage

### 🎯 Supported Skills
- **Installation Sources**: Git repositories, GitHub shorthand (owner/repo), local filesystem paths
- **Agent Targets**: Universal (`.agents/`), Claude (`.claude/`), Cursor (`.cursor/`), Codex (`.codex/`), Gemini (`.gemini/`), Copilot (`.github/`)
- **Scopes**: Project-local and global skill directories with configurable installation targets
- **Skill Discovery**: Automatic skill detection with metadata extraction and validation
